## About
This component provides several customisable Light control components, with a colour picker built using [react-colorful](https://omgovich.github.io/react-colorful/).

## API (Props)
### `entity`
e.g. `light.HA_light_entity`
- A HA Light entity name 

### `variant` (Optional)
- Changes the look of the light component
- Options: [`Accordion`, `Popup`, `SeperatePopups`]

### `overrideSupportColorRGB` (Optional)
- Always shows RGB Colour picker.
<Callout type="error">
    This will not make a non-RGB light support RGB,\
    This is a workaround for [this.](#on-first-load-rgb-colour-picker-will-not-show-when-bulb-is-off)
</Callout>

## Common Issues
### On first load, RGB Colour Picker will not show when bulb is off
When communicating capabilities, when a bulb is off, Home Assistant will not communicate if a light can support colour.\
If you know that light supports colour, you can pass the prop `overrideSupportColorRGB={true}`.\
This will force the UI to show the RGB colour picker, which prevents the sudden appearence of the RGB colour picker when the light is turned on.
